# AI 知识库 · 编码规范 v0.2（精简版）

## 适用范围
- 适用于仓库内所有 `*.py`、`*.ts`、`*.tsx` 源码
- 不适用：`third_party/`、`vendor/`、`dist/`、`build/`、`*.generated.*`
- 子目录有更严格规范时，子目录规范优先

## 要做什么
- Python 必须通过 `black --check`
- TypeScript 必须开启 `strict: true`
- 新增/修改代码不得引入新的 TS 类型错误
- 所有公开函数必须有文档（用途、参数、返回值、错误场景）

## 不做什么
- 不允许业务判断里散落魔法字符串（必须提取到常量或 `enum`）
- 允许的字符串例外：日志文案、UI 文案、测试断言、协议固定字段名
- 不允许 `TODO` / `FIXME` 进入 `main`
- 例外：`TODO(ISSUE-123): ...` 这类可追踪工单格式

## 边界 & 验收
- 变更行单测覆盖率 `>= 80%`
- 仓库总体覆盖率不得下降
- 覆盖率豁免：自动生成代码、纯类型定义、迁移脚本、一次性工具脚本

## 怎么验证
- CI 必须通过：格式检查、lint/类型检查、单测、覆盖率门禁
- 任一硬门禁失败，禁止合并到 `main`
